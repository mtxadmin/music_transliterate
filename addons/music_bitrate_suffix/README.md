Дополнительный скрипт по [обработке имён mp3-файлов](https://github.com/mtxadmin/music_transliterate).

## Предыстория и назначение ##

Битрейт формата MP3 ограничен сверху значением 320 kbps. Этого более чем достаточно для любых практических целей. Да, вопрос немного спорный, и поклонники lossless-форматов с этим не согласятся, но, судя по отзывам и тестам, даже отъявленным аудиофилам с аппаратурой за десятки тысяч долларов очень сложно различить хорошо закодированные 320 kbps MP3 и FLAC.

А вот отличить MP3 битрейта 320 kbps от 256/192/160/128 kbps и прочих - не так сложно даже в обычных бытовых условиях. Также, можно оценивать качество не на слух, а бесстрастным и точным спектроанализатором. Например, в бесплатной программе [Audacity](https://manual.audacityteam.org/man/spectrogram_view.html) или даже [онлайн](https://academo.org/demos/spectrum-analyzer/).

Вот пример плохо закодированного MP3-файла в спектроанализаторе программы Audacity:

![spectrogram](https://github.com/mtxadmin/music_transliterate/blob/main/addons/music_bitrate_suffix/spectrogram.PNG)

В обоих каналах (у нас же стерео, да) видим чёрные провалы сверху. Вот это как раз оно, отсутствие высоких частот. Такой файл будет звучать глуховато, и чем шире полоса, тем хуже.

## Описание скрипта ##

Чтобы сразу было видно, какие файлы хорошо бы заменить на более качественные, и был написан данный скрипт. Он проходится по заданной папке, находит все MP3-файлы и добавляет в конец имени значение их битрейта, если оно меньше 320.

Надо отметить, что не каждый найденный в сети файл с битрейтом в 320 kbps будет качественным и чётким. Возможно, кто-то тупо перекодировал старый 160 в якобы новый 320, и нужные частоты **уже** были потеряны и больше не появятся. Но вот совершенно точно, что в 128 и 160 их в принципе никогда не будет.

То есть, после прогона скрипта, числа в конце имени MP3-файла сразу дают понять, какие файлы хорошо бы перекачать на более качественные.

## Установка ##

Для работы скрипта нужны:
  - Python 3. Качается с официального сайта: https://www.python.org/downloads/ и устанавливается очень просто, Далее-Далее-Далее-Готово.
  - модуль mutagen для работы с ID3-тегами в mp3-файлах. Для установки достаточно написать pip install mutagen в консоли Python
  - ну и сам скачанный этот скрипт music_bitrate_suffix.py

## Использование ##

**Запуск:**<br>
python music_bitrate_suffix.py <путь_к_папке>

Тут обратите внимание, что в Windows разделитель пути - обратный слэш \ , а в Python используется прямой / . Как вариант, можно задублировать обратные слэши ( \ -> \\\\ )<br>

То есть, для запуска в папке D:\Music\ вы даёте команду:<br>
**python music_bitrate_suffix.py D:/Music/**<br>

Для запуска в папке Music локального сервера с именем server1:<br>
**python music_bitrate_suffix.py //server1/Music**
